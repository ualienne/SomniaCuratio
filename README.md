# Somnia Curatio



ФИО: Дядькина Ульяна Андреевна (501133), Тарадай Селена Кирилловна (504500)

Версия SFML: 3.0.2

Стандарт C++: C++20



Консольный roguelike dungeon-crawler, в котором агент Бюро Сновидений

погружается в кошмары пациентов и борется с персонификациями страхов.



### Архитектура зачётных систем

* **Основная большая система — Lucidity / Therapy–Increase.**
Параметр осознанности (0–100%) меняется в ответ на долгое взаимодействие игрока с кошмаром. Влияет на концовку. Реализация: src/systems/ArachnaphobiaScene.cpp.
* **Доп. механика 1 — Диалоги и выборы персонажа.**
На текущий момент прописаны в сценах, но планируется реализация в .json.
* **Доп. механика 2 — Сохраняемый дневник / инвентарь в .txt.**
Реализация в `data/world/Camera.

### Сборка

```bash
cmake -B build
cmake --build build --config Release
./build/bin/SomniaCuratio
```

Зависимости (SFML 3.0.2, tmxlite, nlohmann/json) подтягиваются автоматически
через `FetchContent`. Доустановка не требуется.



### Управление

|Клавиша|Действие|
|-|-|
|WASD / Стрелки|Движение|
|E|Взаимодействие с объектом|
|Esc|Выход|



### Репозитории ассетов

* Спрайты персонажей: `assets/sprites/characters/`
* Карты Tiled: `assets/maps/`

### Почему в файлах Arachnophobia.h/.cpp некоторые сырые указатели нельзя заменить на умные?

Сырые const T* здесь нельзя заменить на умные указатели, потому что мы этими объектами не владеем.
findSpawn() и getIf() — это чужие объекты, библиотека сама их создаёт и удаляет. Если мы обернём их в unique_ptr, он попытается удалить их второй раз — краш.
this в лямбде — лямбда просто смотрит на поле объекта, не владеет им. shared_ptr здесь не нужен и сломает всю архитектуру.
